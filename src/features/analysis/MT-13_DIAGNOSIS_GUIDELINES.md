이 문서는 **관계 튜닝 센터(Relationship Tuning Dashboard)**의 진단 로직 구조, 데이터 산출 방식, 집중 조율(FT-11) 이후의 분석 리포트, 그리고 전체 UX 흐름을 정의합니다.

---

## 0. UX Flow Map (조율 여정)

사용자의 관계 균형을 회복하기 위한 단계별 여정은 다음과 같습니다.

1.  **[MT-13] 진단 (Diagnosis)**: 전체 궤도 요약 및 불균형 관계 식별
2.  **[MT-13] 선택 (Select)**: 조율이 필요한 인물 2명 이상 선택 (혹은 시스템 추천 넛지 클릭)
3.  **[FT-11] 조율 (Tournament)**: 1:1 비교를 통한 심리적 우선순위 확정
4.  **[FT-11 Result] 통찰 (Insight)**: 선택 패턴 분석을 통한 내면 가치 확인 및 관계 점수 반영

## 1. 데이터 산출 로직 (Scoring Logic)

모든 진단은 `useRelationshipStore`에 저장된 실제 관계 데이터를 기반으로 실시간 계산됩니다.

### 1.1 에너지 점유율 (Energy Distribution)
*   **산출 방식**: 각 관계의 `temperature(친밀 온도)`를 가중치로 사용하여 Zone별 합산을 구한 후, 전체 에너지 총합 대비 비중(%)을 계산합니다.
*   **수식**: `(특정 Zone 내 관계들의 온도 합 / 전체 관계 온도 총합) * 100`
*   **의미**: 사용자가 특정 관계 영역에 쏟고 있는 심리적 시간과 감정의 밀도를 나타냅니다.

### 1.2 안정성 점수 (Stability Score)
*   **산출 방식**: 각 Zone의 `현재 에너지 비중`과 시스템이 권장하는 `이상적 비중(Target Ideal)`의 차이(절대값)를 합산하여 100점에서 차감합니다.
*   **수식**: `100 - Σ|실제 비중 - 이상적 비중|`
*   **의미**: 나의 관계 궤도가 최적의 균형 상태에서 얼마나 벗어나 있는지를 직관적으로 보여줍니다.

## 2. 상태 판정 및 동적 문구 (Dynamic Phrasing)

에너지 점유율 계산 결과에 따라 대시보드 상단에 표시되는 진단 메시지가 실시간으로 변경됩니다.

### 2.1 포화 상태 판정 (Saturation Check)
특정 구역의 에너지 점유율이 권장 최대치(`Target Max`)를 초과했는지 여부를 기준으로 합니다.

| 상태 | 판정 근거 (Logic) | 표시 문구 |
| :--- | :--- | :--- |
| **포화** | **Actual % > Target Max** (임의의 Zone 하나라도 해당 시) | **"현재 뇌 내 구역이 포화 상태입니다."** |
| **쾌적** | **Actual % ≤ Target Max** (모든 Zone이 해당 시) | **"뇌 내 구역이 비교적 쾌적한 상태입니다."** |

### 2.2 관계 재조정 필요 알림
현재 점유율이 권장 범위를 벗어난 '불균형 관계'의 개수를 표시합니다.

| 상태 | 판정 근거 (Logic) | 표시 문구 |
| :--- | :--- | :--- |
| **주의** | 불균형 관계 수 > 0 | **"{N}개의 관계를 재조정해야 합니다."** (테라코타색 강조) |
| **안정** | 불균형 관계 수 = 0 | **"모든 관계가 평온하게 유지되고 있습니다."** (그린색 표시) |

---

### 2.3 안정성 지표 (Stability Metrics)
안정성 지표 영역의 뱃지 문구는 다음과 같습니다.

| 메트릭 명칭 | 영문 병기 (참고) | 설명 |
| :--- | :--- | :--- |
| **안정성** | Stability | 궤도의 전반적인 조화로운 정도 (%) |
| **불균형** | Imbalanced | 권장 범위를 이탈한 관계가 존재할 시 표시 |
| **평형 상태** | Balanced | 모든 구역이 권장 범위 내에 존재할 시 표시 |

## 3. 우선 관리(Nudges) 추출 로직

행동 유도를 위해 세 가지 유형의 관계를 우선적으로 추출하여 카드 형태로 제안합니다.

1.  **소홀해진 관계 (Neglected Anchor)**
    *   **대상**: `Zone 1(안전 기지)` 관계 중 마지막 교감 기록(`lastInteraction`)이 가장 오래된 인물.
    *   **의미**: 가장 소중하지만 당연하게 여겨 방치된 관계를 챙기도록 유도.

2.  **에너지 효율 (Energy ROI)**
    *   **대상**: `노력(Input)` 대비 `보람(Output)`이 낮아 심리적 번아웃을 유발할 수 있는 관계.
    *   **의미**: 불필요하게 에너지를 많이 소모하는 관계를 식별하고 조율하도록 제안.

3.  **새로운 인연 (New Orbiter)**
    *   **대상**: 최근 1주일 내에 새롭게 추가되었거나 궤도에 진입한 인물.
    *   **의미**: 새로운 관계가 안정적으로 정착할 수 있도록 관심을 유도.

## 4. 권장 비중 가이드라인 (Reference Targets)

진단의 기준이 되는 `ZONE_INFO` 설정값은 다음과 같습니다.

| 구역명 | 목표 비항 (Ideal) | 적정 범위 (Min~Max) | 비고 |
| :--- | :--- | :--- | :--- |
| **Zone 1: 안전기지** | 20% | 15% ~ 25% | 최우선 지지층 |
| **Zone 2: 성장** | 20% | 15% ~ 25% | 긍정적 시너지 |
| **Zone 3: 확장** | 25% | 20% ~ 30% | 사회적/기능적 관계 |
| **Zone 4: 탐색** | 15% | 10% ~ 20% | 느슨한 연결 |
| **Zone 5: 휴식** | 15% | 10% ~ 20% | 배경 소음 및 거리두기 |

## 5. 집중 조율 결과 분석 (Tuning Ranking & Status)

조율(FT-11)이 완료된 후, 시스템은 토너먼트 우승자를 중심으로 최종적인 관계 우선순위를 재설정합니다.

### 5.1 우승자(1위) 판정 및 의미
*   **Soul Anchor 확정**: 전체 비교에서 최종 승리한 인물은 사용자의 현재 심리적 '안전 기지'로 확정됩니다.
*   **상태 업데이트**: 해당 인물의 `temperature`를 최상단(98~100)으로 고정하거나, Zone 1의 최우선 순위로 배치합니다.

### 5.2 결과 화면 구성 요소 (Ranking Center)
*   **The Winner (오늘의 주인공)**: 1위 우승자를 트로피/왕관 아이콘과 함께 거대하게 노출.
*   **Top 2-3 (상위 랭커)**: 우승자와 경합을 벌인 상위 인물들을 리스트로 표시.
*   **Behind Pattern (선택의 근거)**: 사용자의 무의식적 가치관(안정/성장 등)을 분석하여 제시. (상세 로직은 [FT-11 패턴 분석 로직](./FT-11_PATTERN_ANALYSIS_LOGIC.md) 참조)
*   **Direct Action**: 우승자에게 즉시 메시지를 보내거나 안부를 묻는 버튼 제공.

## 6. 다각도 관계 렌즈 (Multi-Lens Analytics)
각 렌즈는 인맥 궤도를 서로 다른 정서적/물리적 관점에서 분석하여 통찰을 제공합니다.

*   **✨ 영혼의 배터리 (Positive Rank)**: 관계 온도(70도 이상)가 높은 인물을 추출. 정서적 지지와 회복의 중심 축을 식별.
*   **⚠️ 에너지 포식자 (Negative Rank)**: 관계 온도(40도 이하) 혹은 코르티솔 수치가 높은 인물을 추출. 심리적 바운더리 설정이 필요한 대상 식별.
*   **📊 일상의 중력 (Frequency Rank)**: 상호작용 빈도가 가장 높은 인물을 추출. 실제 시간 점유율과 감정적 가치의 불일치를 분석.

### 6.1 분석 렌즈별 스코어링 로직

| 렌즈명 | 핵심 지표 (Criteria) | 분석 목표 |
| :--- | :--- | :--- |
| **✨ 영혼의 배터리** | `(RQS Rank S/A) + (Oxytocin High) + (Temp > 80)` | 나를 회복시키는 핵심 지지층(Anchors) 식별 |
| **⚠️ 에너지 포식자** | `(RQS Rank C) + (Cortisol High) + (Temp < 40)` | 나를 소모시키는 독성 관계(Vampires) 식별 및 방어 |
| **📊 일상의 중력** | `(Interaction Frequency in 7 Days)` | 내 시간을 가장 많이 점유하고 있는 실질적 관계 파악 |

### 6.2 인사이트 매트릭스 (The Matrix)
영향력(Impact)과 빈도(Frequency)를 교차 분석하여 4개의 세그먼트로 관계를 분류합니다.

1.  **[High Impact x High Freq]**: **나의 중심(Center)** - 삶의 질을 결정하는 핵심 인물.
2.  **[High Impact x Low Freq]**: **잃어버린 보석(Hidden Gem)** - 큰 힘이 되지만 방치된 관계. (우선 연락 권장)
3.  **[Low Impact x High Freq]**: **에너지 누수(Energy Leak)** - 자주 보지만 보람 없는 관계. (거리두기 제안)
4.  **[Low Impact x Low Freq]**: **정리 대상(Background)** - Zone 5로 이동하여 배경 소음 처리.

## 7. 맥락 인지형 조율 (Context-Aware Tuning)
분석 단계(Analytics)에서 조율 단계(Action)로 넘어가는 흐름에서 사용자의 의도를 유지하기 위한 이원화된 경로를 제공합니다.

### 7.1 자동 조율 경로 (Contextual Flow)
*   **트리거**: 특정 분석 렌즈(회복, 소모, 빈도)가 활성화된 상태에서 하단의 메인 버튼 클릭.
*   **작동 로직**: 시스템이 현재 렌즈의 분석 점수 기준 상위 6명을 자동으로 후보군으로 추출.
*   **UX 경험**: 별도의 인물 선택 과정을 생략하고 즉시 대진표가 세팅된 토너먼트 화면으로 진입.

### 7.2 수동 조율 경로 (Manual Flow)
*   **트리거**: '균형 요약' 우측의 [직접 선택] 버튼 클릭.
*   **작동 로직**: 사용자가 전체 인맥 궤도 혹은 특정 Zone 필터를 통해 조율하고 싶은 대상(2~8명)을 직접 선택.
*   **UX 경험**: 자신의 직관에 따른 조율 대상을 구성할 수 있는 자율성 보장.
